Edit

Der Edit-Befehl öffnet das aktuelle Skript zum Editieren mithilfe des zugeordneten Verbs "edit", das in der Registry angegeben ist (oder in Notepad, wenn kein Verb vorhanden ist). Wenn das Skript jedoch bereits in einem Editor geöffnet ist (abhängig vom Fenstertitel), wird dieser Editor aktiviert, anstatt eine neue Instanz des Editors zu öffnen.

Dieser Befehl hat keine Wirkung, wenn er innerhalb eines kompilierten Skripts ausgeführt wird.

Beachten Sie, dass bei vielen Editoren eine Syntaxhervorhebung für AutoHotkey aktiviert werden kann (siehe unten). Zusätzlich kann mit diesem Beispiel eine kontextabhängige Hilfe für AutoHotkey-Befehle aktiviert werden. Zur Steigerung der Produktivität kann ein Autovervollständigungs-Tool wie das Skript von boiler oder das Skript von Helgef verwendet werden, das mit fast jedem Editor funktioniert. Es überwacht, was Sie eingeben, und zeigt Menüs und Parameterlisten an, die Ihnen einen Teil der Tipparbeit abnehmen und Sie an die Reihenfolge der Parameter erinnern.

Editoren mit AutoHotkey-Unterstützung

SciTE4AutoHotkey ist eine angepasste Version des Texteditors SciTE, zugeschnitten auf die Editierung von AutoHotkey-Skripten. Es hat folgende Features:

  • Syntaxhervorhebung
  • Intelligente automatische Einrückung
  • Autovervollständigung
  • Calltips (auch bekannt als IntelliSense)
  • Codefaltung
  • Unterstützung für interaktives Debuggen
  • Weitere Tools für AutoHotkey-Scripting

SciTE4AutoHotkey kann hier gefunden werden: https://www.autohotkey.com/scite4ahk/

AHK Studio ist ein via AutoHotkey erstellter Editor zum Editieren von AutoHotkey-Skripten. Weitere Informationen, Video-Anleitungen und eine ständig wachsende Liste von Features finden Sie im folgenden Forum-Thread: AHK Studio

AutoGUI ist eine integrierte Entwicklungsumgebung für AutoHotkey, bestehend aus einem GUI-Designer und einem Skripteditor. Sie können es hier finden: AutoGUI - GUI Designer and Script Editor

Weitere Editoren, für die eine Syntaxhervorhebung für AutoHotkey aktiviert werden kann:

  • AkelPad
  • Crimson Editor
  • Eclipse, FAR-Manager, und andere Editoren, die Colorer take5 verwenden
  • Emacs
  • Kate
  • Notepad++
  • Notepad2
  • PSPad
  • SciTE und möglicherweise andere Scintilla-basierte Editoren
  • Sublime Text Editor
  • Total Commander mit dem Synplus-Plugin
  • Visual Studio Code mit der AutoHotkey-Erweiterung von Mark Wiemer

Zusätzlich enthält die Zip-Version von AutoHotkey Basic (https://www.autohotkey.com/download/1.0/) Dateien zur Aktivierung der Syntaxhervorhebung für die folgenden Editoren. Einige dieser Dateien sind jedoch veraltet und funktionieren möglicherweise nicht mehr:

  • ConTEXT
  • EditPlus
  • EmEditor
  • jEdit
  • MED
  • TextPad
  • UltraEdit
  • Vim
