GroupDeactivate

Verhält sich wie GroupActivate, aber aktiviert das nächste Fenster, das kein Mitglied der Gruppe ist.

GroupDeactivate, GruppeName , Modus

Parameter

GruppeName

Name der Zielgruppe, die ursprünglich mit GroupAdd definiert wurde.

Modus

Lässt man diesen Parameter weg, wird der Befehl das älteste von den nicht zur Gruppe gehörenden Fenstern aktivieren. Um dieses Verhalten zu ändern, geben Sie den folgenden Buchstaben an:

R: Das neueste (zuletzt aktive) von den nicht zur Gruppe gehörenden Fenstern wird aktiviert, aber nur, wenn beim Aufrufen des Befehls ein Fenster der Gruppe aktiv ist. "R" ist zum Beispiel nützlich, wenn Sie vorübergehend zu einer anderen Aufgabe wechseln müssen. Wenn Sie mit GroupActivate, GroupDeactivate oder GroupClose zur Gruppe zurückkehren, wird das Fenster, mit dem Sie zuletzt gearbeitet haben, aktiviert, nicht das älteste Fenster.

Bemerkungen

GroupDeactivate bewirkt, dass das erste Fenster, das nicht mit den Spezifikationen der Fenstergruppe übereinstimmt, aktiviert wird. Verwendet man GroupDeactivate danach nochmals, wird das nächste Fenster in der Reihe aktiviert und so weiter. Normalerweise weist man GroupDeactivate einem Hotkey zu, um dieses Fensterdurchschaltverhalten durch Drücken dieser Taste zu automatisieren.

Dieser Befehl ist zum Beispiel für eine Sammlung von favorisierten Fenstern nützlich, die immer offen bleiben sollen. Wenn Sie diese Fenster zu einer Gruppe hinzufügen, können Sie mit GroupDeactivate jedes Fenster besuchen, das nicht zu Ihren Favoriten gehört, und dann entscheiden, ob Sie dieses Fenster schließen wollen oder nicht. Diese Vorgehensweise erlaubt Ihnen, Ihr Desktop sehr viel schneller aufräumen, als es manuell möglich wäre.

Weitere Informationen zum Thema Fenstergruppen finden Sie unter GroupAdd.
